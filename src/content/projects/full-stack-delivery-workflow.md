## Context

My daily engineering work spans both the frontend and backend side of web applications. I like clear interfaces, predictable builds, and implementations that stay understandable after the first version ships.

## Focus

- Building and maintaining Angular-based user interfaces.
- Working on Java/Spring services with Gradle-based build pipelines.
- Connecting product requirements with practical technical decisions.
- Using AI-assisted exploration to speed up unfamiliar code paths, refactors, and documentation.

## Why it matters

Full-stack work benefits from context. Understanding the frontend, backend, build, and deployment path makes it easier to debug issues and deliver changes that fit the whole system.
